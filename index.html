Project Idea

For the project Idea I would like to develop a timetabling/to-do list smartphone app which allows a user to track there day to day tasks and upcoming events. This app will also provide daily reminders for important occasions such as assignments, meetings and presentations to not only remind the user of what needs to be done but allows them to input a quick progress report to keep up-to-date with their life. The daily progress report/reminder would be a quick rating system that would take less than 10 seconds to input and therefore not be a nuisance to the user. I think this project will be interesting to me as I have never developed a smartphone app before and even though it is a daunting task as I have no experience or idea on where to start, I am very interested in trying new challenges. I also believe that this app will be useful to those who need a planner to keep themselves organised. The concept of a planner type app is not new and is actually commonplace in most starting smartphone app lineups, however I believe with the added feature of a progress report/reminder system is that it will help those stay on top of things more-so because the progress report function will be a constant reminder of how much the user has or hasn’t done encouraging them to stay ahead/on time. The main function of this app is to be able to input tasks, events and schedules too organise and simplify the users lifestyle. This means that there will be a calendar that will allow users to add tasks, events and schedules. This will take form in a ‘+’ icon that gives users options to create the event and add it to a specific day/days. There will also be an option to set tasks and events as recurring which allows the user to create a functioning timetable. There will also be a function that allows users to edit pre-existing tasks. There will also be a reminder function, this function will provide the user with daily reminders that something is upcoming for example, the user has a test due in 5 days starting from a week before that day, there will be daily reminders that keep the user informed that the event is close. Paired with this there will be a progress report system that will work in tandem with the daily reminders. The progress report will be entering roughly how much the user has completed (0% - 100%, 100% being completed) these functions can be turned on/off when setting the tasks. Alongside the progress report there will be an option to also include a checklist that will follow conventional checklist functions. The UI is to minimalistic using icons and buttons that have a universal meaning making it easy for people of all ages to pick up and instantly understand the messages the app is trying to display. Upon opening the app the UI will consist of a calendar with various buttons on the top of the display similar to apple’s calendar UI. The buttons to be included on there are a plus icon for adding tasks, events and schedules, A minus icon to delete pre-existing tasks and a button that says ‘Checklist’ which will open a new UI that allow users to enter checklist entries which they can tick off as they complete them. The colour pattern for the UI is to be a combination of light and shaded colours to give the app a look of simplicity and accessibility to once again communicate to all ages how the app works. The software needed can be one app development program such as MIT app Inventor, Siberian CMS – Open Source App Building, Swiftic or Shoutem. Hardware needed will be computers (Laptops or desktops) for creating the actual app and phones for testing the app and making sure everything works well. Computer hardware requirements are a processor with a minimum of 1GHz, a wifi or ethernet connection, minimum hard drive space of 32Gb and a minimum memory (RAM) of 1Gb. For phones they must be an android device on the latest firmware, with a working touchscreen. The skills required will be analytical skills, communication, creativity, problem solving and programming languages. Analytical skills are important for discerning what the app needs for the target audience of people aged 18 to 60. Communication will be needed to work within a group to achieve the best outcome and to properly communicate ideas and concerns to the group. Creativity and problem solving will be important for tackling the various problems which can arise. Finally programming languages will be important because without them it will be impossible to create an app. I believe it will be easy to obtain the hardware and software needed for this project however obtaining the skills will be the difficult part. With the help of the internet and its sources I do believe however that it can be done even if the final product isn’t fully functioning. If the project is successful then the outcome will be a functioning timetable/planner app that the public can use for free. Everyone involved in the process will also gain valuable knowledge about app development going forward which will help push us closer to obtaining our degree after our 3 years at RMIT. This development will push the group to work harder and potentially even exhaust the group by the end of the project. The original problem will be solved by the app completing everything it was intended to do with zero to a little amount of bugs included.
